# Гайд по использованию спойлеров в постах

## Обзор функционала

Форма создания постов (`CreatePost.tsx`) теперь поддерживает:

1. **Emoji** — вставка emoji через picker, отображаются как картинки в форме
2. **Mention** — упоминание пользователей через `@username`, отображаются как цветные теги
3. **Spoiler** — скрытие текста через выделение + кнопка "Spoiler", текст размывается и раскрывается по клику

## Архитектура

### Токены (Token-based format)

Текст поста хранится как строка с токенами:

- `[emoji:0]`, `[emoji:1]` — emoji (индекс в массиве `emojiUrls`)
- `[mention:userId|userName]` — упоминание пользователя
- `[spoiler]текст[/spoiler]` — спойлер

### Редактор (`contentEditable`)

Вместо `<textarea>` используется `<div contentEditable>` с live-рендером:

- **serializeDOM()** — конвертирует DOM → строку с токенами
- **renderToDOM()** — конвертирует токены → DOM элементы (img, span)
- При вводе текста автоматически рендерятся emoji/mention/spoiler прямо в форме

### Применение спойлера

1. Пользователь выделяет текст в редакторе
2. Появляется кнопка "Spoiler" под выделением
3. При клике выделенный текст оборачивается в `<span data-spoiler="true" class="mc-spoiler">`
4. Спойлер отображается с `filter: blur(5px)` и снимается при клике

### Защита от вложенных спойлеров

- Проверяет, содержит ли выделение уже существующий спойлер
- Проверяет, находится ли выделение внутри спойлера
- Если да — показывает toast-ошибку и не применяет спойлер

## Использование

### В форме создания поста

```tsx
<CreatePost />
```

### В отображении постов

```tsx
<EmojiText text={postContent} emojiUrls={emojiUrls} />
```

Компонент `EmojiText` автоматически:
- Парсит токены через `parseEmojiText()`
- Рендерит emoji как `<img>`
- Рендерит mention как `<Link>` с тултипом пользователя
- Рендерит spoiler через `<SpoilerSegment>` с blur и кликом для раскрытия

## Стили спойлера

```css
.mc-spoiler {
  filter: blur(5px);
  cursor: pointer;
  transition: filter 0.2s;
  background: rgba(0, 0, 0, 0.06);
}

.mc-spoiler.mc-spoiler--revealed {
  filter: none;
}
```

## API отправки поста

```ts
const formData = new FormData();
formData.append("content", postContent); // строка с токенами
formData.append("emojiUrls", JSON.stringify(emojiUrls)); // массив URL
if (selectedImage) {
  formData.append("image", selectedImage);
}
```

Бэкенд должен:
1. Сохранить `content` как есть (с токенами)
2. Сохранить `emojiUrls` как JSON
3. При отдаче поста вернуть оба поля

## Troubleshooting

### Спойлер не применяется

- Проверьте консоль на ошибки
- Убедитесь, что выделение не содержит уже существующий спойлер
- Попробуйте выделить текст заново

### Сайт зависает при применении спойлера

- **Исправлено:** добавлена защита от вложенных спойлеров и проверка родителей
- Если всё равно зависает — проверьте, не содержит ли выделение частично вложенные элементы (например, половину mention-тега)

### Спойлер не отображается в посте

- Проверьте, что бэкенд возвращает `emojiUrls` для поста
- Убедитесь, что `EmojiText` используется для рендера контента
- Проверьте, что `parseEmojiText()` корректно парсит `[spoiler]...[/spoiler]`
